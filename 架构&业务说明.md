# Jeepay（计全支付）完整业务逻辑说明文档

## 文档说明

本文档详细描述 Jeepay（计全支付）系统的业务逻辑和架构设计，涵盖支付网关、运营平台、商户系统三大核心模块的完整业务流程。

## 一、系统整体架构

### 1.1 系统定位

Jeepay 是一套开源的聚合支付系统，支持多渠道服务商和普通商户模式，可对接微信支付、支付宝、云闪付等多种支付渠道。

### 3.3 支付订单核心代码

```java
/**
 * 生成支付订单
 */
private PayOrder genPayOrder(UnifiedOrderRQ rq, MchInfo mchInfo, MchApp mchApp,
                              String ifCode, MchPayPassage mchPayPassage) {
    PayOrder payOrder = new PayOrder();
    payOrder.setPayOrderId(SeqKit.genPayOrderId());     // 生成订单ID
    payOrder.setMchNo(mchInfo.getMchNo());              // 商户号
    payOrder.setIsvNo(mchInfo.getIsvNo());              // 服务商号
    payOrder.setMchName(mchInfo.getMchShortName());     // 商户名称
    payOrder.setMchType(mchInfo.getType());             // 商户类型
    payOrder.setMchOrderNo(rq.getMchOrderNo());         // 商户订单号
    payOrder.setAppId(mchApp.getAppId());               // 应用ID
    payOrder.setIfCode(ifCode);                         // 接口代码
    payOrder.setWayCode(rq.getWayCode());               // 支付方式
    payOrder.setAmount(rq.getAmount());                 // 订单金额
    payOrder.setMchFeeRate(mchPayPassage.getRate());    // 手续费率
    payOrder.setMchFeeAmount(AmountUtil.calPercentageFee(
        payOrder.getAmount(), payOrder.getMchFeeRate()));// 商户手续费
    payOrder.setState(PayOrder.STATE_INIT);             // 订单状态
    payOrder.setDivisionMode(rq.getDivisionMode());     // 分账模式

    // 订单过期时间（默认2小时）
    if(rq.getExpiredTime() != null){
        payOrder.setExpiredTime(DateUtil.offsetSecond(new Date(), rq.getExpiredTime()));
    } else {
        payOrder.setExpiredTime(DateUtil.offsetHour(new Date(), 2));
    }

    return payOrder;
}
```

### 4.2 回调处理核心代码

```java
/**
 * 异步回调处理
 */
@ResponseBody
@RequestMapping("/api/pay/notify/{ifCode}")
public ResponseEntity doNotify(HttpServletRequest request,
                               @PathVariable("ifCode") String ifCode) {

    // 1. 获取支付接口实现
    IChannelNoticeService payNotifyService = SpringBeansUtil.getBean(
        ifCode + "ChannelNoticeService", IChannelNoticeService.class);

    // 2. 解析回调参数
    MutablePair<String, Object> mutablePair = payNotifyService.parseParams(
        request, urlOrderId, IChannelNoticeService.NoticeTypeEnum.DO_NOTIFY);

    String payOrderId = mutablePair.left;

    // 3. 查询订单
    PayOrder payOrder = payOrderService.getById(payOrderId);

    // 4. 获取商户配置
    MchAppConfigContext mchAppConfigContext = configContextQueryService.queryMchInfoAndAppInfo(
        payOrder.getMchNo(), payOrder.getAppId());

    // 5. 执行回调处理
    ChannelRetMsg notifyResult = payNotifyService.doNotice(
        request, mutablePair.getRight(), payOrder, mchAppConfigContext,
        IChannelNoticeService.NoticeTypeEnum.DO_NOTIFY);

    // 6. 更新订单状态（仅处理支付中的订单）
    if(payOrder.getState() == PayOrder.STATE_ING) {
        if(ChannelRetMsg.ChannelState.CONFIRM_SUCCESS == notifyResult.getChannelState()) {
            // 支付成功
            payOrderService.updateIng2Success(payOrderId,
                notifyResult.getChannelOrderId(), notifyResult.getChannelUserId());
        } else if(ChannelRetMsg.ChannelState.CONFIRM_FAIL == notifyResult.getChannelState()) {
            // 支付失败
            payOrderService.updateIng2Fail(payOrderId,
                notifyResult.getChannelOrderId(), notifyResult.getChannelUserId(),
                notifyResult.getChannelErrCode(), notifyResult.getChannelErrMsg());
        }
    }

    // 7. 支付成功后续处理
    if(notifyResult.getChannelState() == ChannelRetMsg.ChannelState.CONFIRM_SUCCESS) {
        payOrderProcessService.confirmSuccess(payOrder);
    }

    return notifyResult.getResponseEntity();
}
```

## 六、分账业务

### 6.1 分账模式

```
分账模式 (division_mode):
├── 0 - 不允许分账 (DIVISION_MODE_FORBID)
├── 1 - 自动分账 (DIVISION_MODE_AUTO)
│   └── 支付成功后自动触发分账
└── 2 - 手动分账 (DIVISION_MODE_MANUAL)
    └── 商户手动发起分账请求

分账状态 (division_state):
├── 0 - 未发生分账 (DIVISION_STATE_UNHAPPEN)
├── 1 - 等待分账任务处理 (DIVISION_STATE_WAIT_TASK)
├── 2 - 分账处理中 (DIVISION_STATE_ING)
└── 3 - 分账任务已结束 (DIVISION_STATE_FINISH)
```

```java
/**
 * 验签处理
 */
protected <T> T getRQByWithMchSign(Class<T> cls) {
    // 1. 获取请求参数
    String reqJsonStr = getReqParamFromBody();

    // 2. 解析为对象
    T rq = JSONObject.parseObject(reqJsonStr, cls);

    // 3. 获取商户应用配置
    String mchNo = getValStringRequired("mchNo");
    String appId = getValStringRequired("appId");
    MchApp mchApp = configContextQueryService.queryMchApp(mchNo, appId);

    // 4. 验证签名
    String sign = getValStringRequired("sign");
    String currentSign = JeepayKit.getSign(getReqParamMap(), mchApp.getAppSecret());

    if (!sign.equals(currentSign)) {
        throw new BizException("签名验证失败");
    }

    return rq;
}
```

## 十八、常见问题处理

### 18.1 订单超时处理

```
订单超时场景:
├── 用户下单后未完成支付
├── 支付渠道处理超时
└── 网络问题导致支付中断

处理方案:
1. 订单创建时设置过期时间 (默认2小时)
2. 定时任务扫描过期订单
3. 更新订单状态为关闭
4. 如已支付，触发补单处理
```

### 18.2 重复支付处理

```
重复支付原因:
├── 用户重复提交支付请求
├── 支付渠道重复回调
└── 网络重试导致的重复

防护机制:
1. 商户订单号唯一索引
2. 订单状态幂等性控制
3. 回调去重处理
4. 分布式锁防护
```

### 18.3 退款失败处理

```
退款失败常见原因:
├── 原支付订单不存在
├── 退款金额超过可退金额
├── 退款时间超限
├── 渠道账户余额不足
└── 银行系统异常

处理方案:
1. 自动重试机制
2. 人工介入处理
3. 线下退款补偿
```
