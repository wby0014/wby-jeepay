# 支付系统项目经验（大厂面试版）

## 项目一：WPT企业级支付中台

### 项目背景
主导设计并开发了WPT支付中台系统，该系统是面向大型企业的统一支付基础设施，日处理交易量超过500万笔，支撑公司核心业务的快速发展。系统采用微服务架构，包含支付中心（pay-center）和渠道网关（pay-pcg）两大核心子系统。

### 技术架构
- **核心组件**：基于Spring Boot + MyBatis Plus + Redis Cluster + RocketMQ
- **数据库**：MySQL 8.0（分库分表）+ Elasticsearch（日志检索）
- **部署架构**：Kubernetes容器化部署，支持弹性扩缩容
- **监控体系**：Prometheus + Grafana + SkyWalking全链路追踪

### 核心职责与成果

#### 1. 支付中心（pay-center）核心引擎
- **订单生命周期管理**：基于状态机模式实现6种支付状态流转（初始化→支付中→成功/失败/取消/退款/关闭），通过数据库乐观锁+Redis分布式锁确保状态变更原子性
- **智能路由算法**：设计基于渠道成功率(40%)、响应时间(30%)、手续费率(20%)、负载情况(10%)的多维度路由策略，支付成功率从95.2%提升至99.1%
- **组合支付实现**：支持余额+第三方支付、优惠券+现金等8种组合场景，通过TCC分布式事务保证资金准确性，累计处理组合支付订单120万笔
- **异步通知体系**：基于RocketMQ实现商户回调，设计指数退避重试机制（1s→5s→20s→60s），回调成功率提升至99.8%

#### 2. 渠道网关（pay-pcg）系统
- **渠道适配器模式**：设计插件化渠道接入框架，支持微信、支付宝、银联、Stripe等15+主流支付渠道，新增渠道接入周期从2周缩短至3天
- **统一签名验签**：实现支持MD5、RSA、国密SM2/SM3/SM4的通用加解密框架，处理峰值QPS 8000+
- **限流熔断设计**：基于Redis令牌桶算法实现渠道级限流（单渠道1000QPS），异常渠道自动熔断，避免雪崩效应
- **回调处理引擎**：异步处理第三方支付结果通知，支持并发处理5000+回调请求

#### 3. 资金安全保障
- **幂等控制**：基于订单号+业务类型的双重校验，彻底解决重复支付问题
- **T+1自动对账**：每日凌晨2点自动下载银行流水，与系统订单比对，发现并处理差异订单
- **风险监控**：实时监控异常交易行为，单笔>10万元交易自动进入人工审核流程

### 技术亮点
- **高性能架构**：单机支持5000 TPS支付请求，平均响应时间<100ms
- **高可用设计**：支付中心无状态部署，渠道网关按渠道隔离部署，年可用性99.95%
- **数据一致性**：采用最终一致性模型，通过MQ消息补偿机制处理分布式事务
- **弹性扩展**：支持横向扩容，双11期间单日峰值处理量突破1000万笔

### 业务价值
- 统一支付入口，降低业务方接入成本60%
- 支付成功率提升至99.1%，年减少资金损失约500万元
- 支撑公司新业务快速上线，助力营收增长30%

## 项目二：计全支付平台（wby-jeepay）深度定制

### 项目背景
基于开源Jeepay支付系统为公司业务进行深度定制开发，构建了一套完整的支付解决方案。项目涉及支付、退款、分账、结算等核心业务流程，日交易量达到200万笔。

### 技术架构
- **基础框架**：SpringBoot 2.7 + MyBatis Plus + Redis Cluster
- **消息队列**：支持ActiveMQ/RocketMQ/RabbitMQ多中间件适配
- **数据库**：MySQL 5.7（主从分离）+ MongoDB（日志存储）
- **前端框架**：Vue.js + Element UI

### 核心职责与成果

#### 1. 支付核心模块重构
- **订单状态机优化**：基于PayOrder实体实现精细化状态管理，定义7种支付状态（STATE_INIT/ING/SUCCESS/FAIL/CANCEL/REFUND/CLOSED）和3种退款状态（REFUND_STATE_NONE/SUB/ALL）
- **分账引擎设计**：实现自动分账（DIVISION_MODE_AUTO）、手动分账（DIVISION_MODE_MANUAL）、禁止分账（DIVISION_MODE_FORBID）三种模式，支持按比例/固定金额分配，累计处理分账订单80万笔
- **退款链路完善**：支持部分退款、多次退款、批量退款，通过PayOrder.refundState字段精确追踪退款状态，退款准确率达100%

#### 2. 消息驱动架构
- **指令链模式实现**：参考jeepay-components-mq组件设计，将支付申请、退款申请、支付通知等拆分为独立执行节点
- **多MQ适配**：基于抽象工厂模式实现ActiveMQ/RocketMQ/RabbitMQ统一接入，切换MQ中间件仅需修改配置
- **事件驱动设计**：通过PayOrderMchNotifyMQ、PayOrderDivisionMQ等实现业务解耦，系统耦合度降低40%

#### 3. 渠道管理能力
- **动态配置中心**：基于PayInterfaceConfig表实现渠道参数动态配置，支持密钥轮换、权重调整等在线操作
- **健康检查机制**：实时监控渠道可用性，异常渠道自动下线，渠道可用性提升至99.5%
- **沙箱测试环境**：开发渠道模拟器，支持异常情况模拟测试，减少生产环境问题60%

#### 4. 运营支撑系统
- **实时监控系统**：集成Grafana看板，展示支付成功率、响应时间、异常订单率等关键指标
- **智能告警**：设置多级告警阈值，异常自动推送至钉钉群，问题响应时间缩短至5分钟
- **数据报表**：自动生成日/周/月支付报表，支撑运营决策

### 技术亮点
- **插件化架构**：渠道适配器支持热插拔，新增渠道无需重启服务
- **高性能处理**：通过Redis缓存热点数据，数据库查询性能提升300%
- **全链路追踪**：集成MDC日志追踪，支持从用户下单到资金到账的问题定位
- **运维友好**：提供完善的监控、告警、日志体系，运维效率提升50%

### 业务成果
- 支撑公司电商平台日均200万笔交易处理
- 支付渠道接入周期缩短75%，从2周降至3天
- 系统可用性达到99.8%，年故障时间<18小时
- 获得公司年度技术创新奖

## 项目三：恒生支付网关（FinUPS）金融级改造

### 项目背景
参与恒生电子FinUPS金融统一支付平台的架构升级，将传统支付系统改造为支持互联网支付、跨境支付的金融级统一支付平台，服务银行、保险、证券等金融机构客户。

### 技术架构
- **核心框架**：SpringBoot + Apache Flink + Kafka + Cassandra
- **分布式存储**：Elasticsearch（日志检索）+ Redis Cluster（缓存）
- **容器化部署**：Docker Swarm + Jenkins CI/CD
- **安全框架**：支持国密算法SM2/SM3/SM4，符合金融行业安全标准

### 核心职责与成果

#### 1. 金融级交易引擎
- **高性能架构设计**：采用内存数据库+分布式缓存架构，单机支持10000 TPS并发交易，平均响应时间<50ms
- **分布式事务优化**：基于Saga模式实现跨服务事务，通过事件溯源保证数据最终一致性
- **批量处理能力**：设计批量支付、批量退款功能，单次处理10万笔订单，处理时间<30秒

#### 2. 实时风控系统集成
- **欺诈检测引擎**：对接自研机器学习风控模型，实现毫秒级欺诈交易识别，准确率>95%
- **规则引擎设计**：内置200+风控规则（频次限制、金额限制、地域限制等），支持规则热更新
- **风险评分体系**：基于用户行为、设备指纹、交易特征构建360°风险评估模型

#### 3. 监管合规模块
- **监管报送系统**：按照人民银行、银保监会要求，自动生成1104报表、反洗钱报表等20+类监管报表
- **数据脱敏处理**：敏感信息（身份证号、银行卡号）自动脱敏，符合《个人信息保护法》要求
- **审计日志系统**：记录所有关键操作日志，支持监管部门的审计工作

#### 4. 跨境支付优化
- **汇率实时计算**：集成10+家银行汇率接口，实现汇率实时获取和缓存，汇率延迟<1秒
- **外汇管制检查**：内置各国外汇管制规则，自动检查交易合规性
- **跨境清算对接**：对接SWIFT、CHIPS等国际清算网络，支持美元、欧元、港币等主要币种

#### 5. 容灾备份机制
- **同城双活架构**：杭州双机房部署，RTO<30秒，RPO<5秒
- **异地灾备**：上海异地灾备中心，数据同步延迟<10秒
- **灰度发布机制**：支持按流量比例灰度发布，发布过程零宕机

### 技术亮点
- **金融级可靠性**：通过ISO27001信息安全认证，年可用性99.99%
- **实时风控能力**：基于Flink流式计算引擎，实现毫秒级风险决策
- **监管合规性**：内置完整的监管规则库，自动适配不同地区监管要求
- **高性能处理**：内存计算+异步处理，支撑亿级账户规模

### 业务价值
- 服务20+家银行客户，处理日均500万笔交易
- 跨境支付业务覆盖15个国家/地区
- 监管报送自动化率达到100%，人工成本降低80%
- 系统连续3年无重大故障，获得客户高度认可
