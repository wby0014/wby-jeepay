# 支付核心中台与渠道网关系统

## 项目描述

基于 `pay-center` 与 `pay-pcg` 构建平台级支付基础设施，承担交易支付、退款、提现、结算、分账、转账、账务处理及多支付渠道接入能力。项目采用分层架构、指令链编排和事件驱动设计，面向复杂支付场景提供统一交易入口与渠道网关能力，支撑支付主链路的高可用、可扩展与资金处理闭环。

该项目虽然基于 PHP/Lumen 技术栈落地，但核心能力集中在支付域建模、交易链路设计、资金状态流转、异步解耦、渠道抽象和一致性治理，与资深 Java 支付/收单岗位关注的问题高度一致。

## 技术栈

PHP、Lumen、MySQL、Redis、Queue、事件驱动、指令链模式、支付渠道 SDK 对接

## 项目职责与亮点

- 负责支付核心中台与渠道网关的核心能力建设，围绕支付、退款、提现、结算、分账、转账等场景设计统一交易编排链路，推动支付能力从单点接口开发演进为平台化服务。
- 参与交易核心模块设计，基于指令链模式拆分支付申请、支付通知、退款申请、结算申请等多个执行环节，提升复杂交易流程的可编排性、可扩展性和问题定位效率。
- 负责统一支付能力建设，支持普通支付、组合支付、余额支付及多订单合并支付，完成支付方式路由、支付方式解析与排序等核心逻辑，降低不同业务线接入支付能力的复杂度。
- 参与退款链路建设，支持单笔退款、批量退款、部分退款及多次退款等场景，重点处理原订单状态校验、退款金额边界控制、重复退款防护和渠道退款回调后的状态推进，保障退款资金安全与流程可追溯。
- 参与提现与结算能力建设，覆盖提现申请、签约校验、额度控制、渠道出款、异步回调、结算金额计算、手续费处理等关键环节，形成从账户校验到资金出款再到账务落账的完整闭环。
- 参与账务与资金侧能力建设，围绕账户服务、账户额度、交易流水、账务调整等模块处理资金变更和状态同步问题，提升支付、退款、提现等多链路下的账务一致性和可审计性。
- 负责支付渠道网关能力建设，抽象微信支付、支付宝、连连支付、易宝支付、Stripe、PayPal 等渠道接入模式，统一处理下单请求构建、参数校验、签名验签、通知解析、结果分发和失败重试，降低新增渠道接入成本。
- 参与第三方支付通知链路治理，围绕支付、退款、提现等异步回调场景设计标准化处理流程，重点解决通知验签解密、重复通知幂等、业务状态推进、通知下游失败补偿等问题，提升渠道回调链路稳定性。
- 参与支付系统可扩展能力建设，在渠道层和中台层之间形成清晰职责边界，使业务编排与渠道适配解耦，支持后续快速扩展国际卡支付、钱包支付及其他本地支付方式。
- 参与支付对账与账单处理相关能力建设，通过渠道账单下载、账单解析、异常账单处理与提现流水记录等机制，增强支付结果核对和异常问题排查能力。

## 面向资深 Java 支付/收单岗位的价值表达

- 具备完整支付主链路经验，覆盖统一下单、支付结果通知、订单状态流转、退款、提现、结算、分账、账务处理等核心环节。
- 具备多渠道收单接入经验，理解渠道差异抽象、统一网关封装、签名验签、异步通知、失败补偿等收单系统关键问题。
- 具备资金安全与一致性意识，能够围绕幂等、状态机、边界校验、异步解耦和审计追踪设计稳定的支付后端系统。
- 具备中台化建设经验，能够从单业务需求抽象出统一能力，支撑多业务线、多支付方式和多渠道场景复用。
- 具备与国际化收单岗位相关的可迁移经验，已接触 Stripe、PayPal 等海外支付渠道，理解渠道接入标准化、开放接口、风控审核和退款闭环等典型问题。
